# 素材标准化规范（6 类文件）

本规范用于把不同业务域的素材文件稳定抽取为 `meta/` 知识库文件，供高级问数 Text2SQL 引擎使用。

## 1. 输入范围（与文件 6 保持一致）

- 文件 1：数据字典（xlsx）——表名、字段名、类型、含义/注释
- 文件 2：码表内容（md/txt）——编码→名称/状态
- 文件 3：样例 SQL（md/sql/txt）——JOIN 模式、过滤逻辑、指标写法
- 文件 4：指标口径知识库（xlsx）——指标定义、统计逻辑、粒度、限制条件
- 文件 5：需求文件（md）——输出字段、维度、指标、注意事项
- 文件 6：需求说明及输出要求（md）——输出格式、依据标注、冲突规则

## 2. 文件命名约定（建议）

- 数据字典：文件名包含 `数据字典`（例如：`1.数据字典.xlsx`）
- 指标口径：文件名包含 `指标口径`（例如：`4.指标口径知识库.xlsx`）
- 码表：文件名包含 `码表`
- 样例 SQL：文件名包含 `样例` 或 `sample` 或文件名为 `3.样例sql.md`

## 3. Excel 表头建议（用于提高命中率）

### 数据字典（xlsx）

推荐表头（至少包含“表名、字段名”两列）：

- 表名 / TABLE_NAME
- 字段名 / 字段名称 / COLUMN_NAME
- 类型 / 字段类型
- 含义 / 字段含义 / 注释 / 备注

### 指标口径知识库（xlsx）

推荐表头（至少包含“指标名称”一列）：

- 指标名称 / 指标名
- 英文名
- 业务含义
- 计算逻辑 / 口径 / SQL
- 来源表
- 统计粒度
- 限制条件

## 4. 输出（meta/）

- `schema.md`：数据字典（表/字段/含义）+（可选）从样例 SQL 提取的表关联关系
- `metrics.md`：指标口径库
- `code_tables.md`：码表
- `sample_sql.md`：样例 SQL
