## 移网用户核心定义

| 条件 | 含义 | 来源 |
|------|------|------|
| `SUBSTR(SERVICE_TYPE,1,2) IN ('40','50')` | 移网用户（移动电话业务） | 需求文档 5.1 / 5.2 |
| `IS_IOT = '0'` | 排除物联网用户 | 需求文档 5.1（黑名单定义） |
| `IS_STAT = '1'` | 仅统计用户 | 需求文档 5.1 |

## 核心指标口径

| 指标 | 口径 | 主表字段 |
|------|------|----------|
| 移网在网用户数 | IS_INNET='1' 的去重用户数 | `COUNT(DISTINCT USER_ID) WHERE IS_INNET='1'` |
| 移网新发展用户数 | IS_THIS_DEV='1' 的去重用户数 | `COUNT(DISTINCT USER_ID) WHERE IS_THIS_DEV='1'` |
| 移网出账用户数 | IS_ACCT='1' 的去重用户数 | `COUNT(DISTINCT USER_ID) WHERE IS_ACCT='1'` |
| 移网三无用户数 | 通话/短信/流量均低于阈值 | 参见样例 SQL 中 USE_STATUS 定义 |
| 移网活跃用户数 | 有有效使用行为 | 参见样例 SQL 中行为标志 |

## 关联表与码表

| 表名 | 用途 |
|------|------|
| `DWA_V_M_CUS_CB_USER_INFO` | 用户信息宽表（主表） |
| `DWA_V_M_CUS_CB_OM_DATUM` | 融合群组与成员表（判断融合/单用户） |
| `DIM_PROV` | 省分维度码表（prov_id → 省名） |
| `DIM_PRODUCT_WLW` | 物联网产品维表 |
| `DIM_PRODUCT_PRIVACY_NUMBER` | 隐私号产品维表 |
| `DWA_V_M_CUS_CB_SING_USE` | 单用户用量表（三无/活跃判定） |

## 分档维度（用于分档月报场景）

| 维度 | 判定逻辑 |
|------|----------|
| 融合类型 | COMP_ID 空 → 单用户；非空 → 融合用户 |
| 行短标志 | `NET_TYPE_ID='65' AND BRAND_ID_CBSS IN ('HCA1','HDA1')` |
| 隐私号标志 | 关联 DIM_PRODUCT_PRIVACY_NUMBER 有匹配记录 |
| 物联网标志 | `IS_IOT='1'` 或产品在 DIM_PRODUCT_WLW 中或号段为 1454/1455/1456/1457/146 |

## 工作流（5 步）

### 步骤 1：识别移网用户筛选条件
- `SUBSTR(SERVICE_TYPE,1,2) IN ('40','50')`
- `IS_IOT='0'`
- `IS_STAT='1'`
- `MONTH_ID = '${账期}'`

### 步骤 2：对齐需求列与指标口径
- 逐列检查需求文档中的输出字段
- 在指标口径知识库中定位对应定义
- 生成字段对照表（需求列名称 | 口径来源 | 关联字段/计算逻辑）

### 步骤 3：确定主表与关联表
- 主表：DWA_V_M_CUS_CB_USER_INFO
- 按需关联：融合表、省分维表、物联网维表、隐私号维表
- 使用 LEFT JOIN 保证主表数据完整

### 步骤 4：生成 HiveSQL
- 按样例 SQL 风格编写
- 逐行注释标注依据文件与字段来源
- 使用 `COUNT(DISTINCT USER_ID)` 防重复计数

### 步骤 5：校验与方言适配
- 字段顺序与需求文件一致
- 口径完整性检查
- Hive/MaxCompute 方言适配
